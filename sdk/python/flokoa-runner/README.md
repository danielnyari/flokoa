# flokoa-runner

The flokoa generic runner: the operator-deployed runtime that turns a compiled
pydantic-ai `AgentSpec` into a served A2A agent. Most flokoa users never build
a container image — the operator compiles their Agent CR composition into one
spec file and this package does the rest.

The operator ↔ runner interface is normative and versioned: see
[`docs/reference/runtime-contract.md`](../../../docs/reference/runtime-contract.md).

## The pinned platform

Each runner release pins exactly one pydantic-ai version (and its baseline
libraries) in `pyproject.toml`; `runner.lock` is the exported, published
lockfile and `runner-manifest.json` is its machine-readable identity (baked
into the image at `/etc/flokoa/runner-manifest.json`). The AgentSpec JSON
schema for the pinned version is generated by `hack/gen_agentspec_schema.py`
and embedded into the operator binary, which validates every compiled spec
against it before a pod ever starts.

Regenerate all three coherently from `sdk/python/`:

```bash
make runner-contract
```

CI fails on drift between the pin, the lockfile, the manifest, and the
embedded schema. `pydantic-ai-harness` is deliberately not part of the
baseline (capability artifacts own it); CI asserts it never appears in
`runner.lock`.
